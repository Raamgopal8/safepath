# Security Policy

## Reporting Security Vulnerabilities

We take the security of our project seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Reporting Process

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, please report security vulnerabilities by emailing the project maintainers at:

- [Your Security Contact Email]

You should receive a response within 48 hours. If for some reason you do not, please follow up via email to ensure we received your original message.

### Guidelines for Reporting

Please include the following information in your report:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes or mitigations

### Responsible Disclosure

We kindly ask that you:

- Give us reasonable time to fix the issue before making it public
- Avoid accessing or modifying user data
- Do not perform DoS attacks or degrade the service

### Recognition

We appreciate your help in keeping our project secure. With your permission, we will acknowledge your contribution in our security advisory.

## Security Updates

Security updates will be released as soon as possible after a vulnerability has been confirmed and fixed. We will communicate updates through:

- GitHub Security Advisories
- Release notes
- Direct communication with reporters

Thank you for helping make our project safer!
