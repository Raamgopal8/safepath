## Contributing to Safe Path 🚀

Thank you for your interest in contributing to **Safe Path**!  
This project welcomes **beginners, students, and experienced developers**.

This guide explains **everything step by step in simple English**, so anyone can follow it easily 😊

---

## 🌟 Who Can Contribute?
Anyone can contribute:
- Beginners learning open source
- Students
- Developers
- UI/UX designers

No prior open-source experience is required 😊

---

## 🛠️ What You Can Contribute
You can contribute by:
- Improving UI and CSS
- Fixing bugs
- Improving documentation
- Adding new features
- Making the website responsive
- Improving content clarity

---

## 📌 Step-by-Step Contribution Guide (Beginner Friendly)

### 🔹 Step 1: Fork the Repository
1. Open the main CryptoHub repository on GitHub
2. Click the **Fork** button (top-right corner)
3. A copy of the repository will be created in your account

---

### 🔹 Step 2: Clone Your Fork
Clone the repository to your local machine:

```bash
git clone https://github.com/<your-username>/<repo-name>.git
cd <repo-name>
````

Add upstream (original repository, only once per project):

```bash
git remote add upstream https://github.com/<original-owner>/<repo-name>.git
```

---

### 🔹 Step 3: Install Dependencies and Expo Mobile Client 
 
Install all required packages:

```bash
npm install
```

Wait until all dependencies are installed successfully.

---
Install Expo Mobile Client on your phone from the app store.

### 🔹 Step 4: Run the Project on Localhost

Start the project locally:

```bash
npm run start
```

Open your camera and scan the QR code shown in the terminal
(example: `http://localhost:8081`)

Make sure the project runs correctly without errors in Mobile Expo Client.

---

### 🔹 Step 5: Create a New Branch

Always create a new branch before making changes:

```bash
git checkout -b your-branch-name
```

Example:

```bash
git checkout -b blog-improvement
```

---

### 🔹 Step 6: Make Your Changes

* Open the project in VS Code
* Make your changes (UI, CSS, content, etc.)
* Save all files

---

### 🔹 Step 7: Test the Project Again

After making changes, make sure:

* No errors appear in terminal
* UI works correctly
* Changes look good

You can run again if needed:

```bash
npm run dev
```

---

### 🔹 Step 8: Add and Commit Changes

Add all changes:

```bash
git add .
```

Commit with a clear message:

```bash
git commit -m "Improved UI Design / New Feature Added"
```

---

### 🔹 Step 9: Push Your Changes

Push your branch to GitHub:

```bash
git push origin your-branch-name
```

---

### 🔹 Step 10: Create a Pull Request (PR)

1. Go to the original repository
2. Click **Compare & Pull Request**
3. Add:

   * Clear title
   * Description of changes
   * Before & After screenshots (if UI related)
4. Submit the PR 🎉

---

## 🔄 Sync Your Fork with Main Repository

If the main repo has new updates, sync your fork:

```bash
git checkout main
git fetch upstream
git merge upstream/main
git push origin main
```

---

## 🧾 Contribution Rules

* Do not push directly to `main`
* One feature or fix per PR
* Write clear commit messages
* Be respectful in communication
